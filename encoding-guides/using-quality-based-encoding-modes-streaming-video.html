Introduction

Lors de l'encodage des sorties vidéo pour le streaming, nous vous déconseillons d'utiliser qualité réglage seul (cela utilise le mode de codage CRF). Si vous préférez utiliser le mode ciblé sur la qualité, nous vous suggérons fortement d'utiliser les paramètres VBV pour limiter la variation du débit binaire afin de se conformer aux spécifications de format de streaming.

Paramètres VBV

  • decoder_bitrate_cap
  • decoder_buffer_size

Autres paramètres recommandés à prendre en compte

  • video_reference_frames
  • video_bframes

Exemple

Voici un exemple d'encodage ciblé de qualité avec des restrictions VBV.

{
    "type":"segmented",
    "width":640,
    "height":360,
    "quality":3,
    "video_codec_profile":"main",
    "video_reference_frames": 4,
    "video_bframes": 3,
    "decoder_bitrate_cap": 1440,
    "decoder_buffer_size": 1920,
    "keyframe_rate": 0.5,
    "max_frame_rate": 30,
    "audio_quality":5,
    "segment_seconds":10
}

Remarques

  • Prière de se référer à Streaming et listes de lecture HTTP en direct (segmenté) et Diffusion HTTP en direct avec MP4 fragmenté pour plus de détails sur les sorties et listes de lecture HLS.
  • Faire référence à Diffusion DASH et listes de lecture pour plus de détails sur les joueurs DASH.

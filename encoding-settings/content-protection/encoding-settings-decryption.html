Vue d'ensemble

Zencoder prend en charge le traitement des fichiers multimédias chiffrés en utilisant le paradigme « chiffrement au repos ». Nous récupérons les fichiers chiffrés de votre serveur et ne les déchiffrons que pendant le traitement. La version déchiffrée du fichier est supprimée immédiatement après le traitement, ce qui garantit la sécurité de vos données.

Nous prenons en charge les fichiers cryptés AES-128 et AES-256 en modes CBC et CTR. Les paramètres de déchiffrement doivent être inclus avec votre demande de travail. Le déchiffrement est activé lorsqu'un decryption_key ou decryption_key_url est spécifié.

decryption_method

decryption_method:String

Versions de l'API : V1, V2

Parent : entrée

Par défaut : aes-128-cbc (si decryption_key ou decryption_key_url sont définis)

Valeurs valides : « aucun », « aes-128-cbc », « aes-128-ctr », « aes-256-cbc » ou « aes-256-ctr »

Types de tâches compatibles : vod

Exemple : aes-128-cbc

Description :

Définissez l'algorithme de décryptage à utiliser pour le décryptage.

  {
    "input": "s3://zencodertesting/test.mov",
    "decryption_method": "aes-128-cbc"
  }

decryption_key

decryption_key:String

Versions de l'API : V1, V2

Parent : entrée

Valeurs valides : Une chaîne hexadécimale de 16 octets (32 caractères, préfixe "0x" facultatif)

Types de tâches compatibles : vod

Exemple : 1234567890abcdef1234567890abcdef

Description :

Définissez la clé de déchiffrement à utiliser pour une entrée chiffrée.

  {
    "input": "s3://zencodertesting/test.mov",
    "decryption_key": "1234567890abcdef1234567890abcdef"
  }

decryption_key_url

decryption_key_url:String

Versions de l'API : V1, V2

Parent : entrée

Valeurs valides : Une URL vers un fichier contenant une clé binaire de 16 octets

Types de tâches compatibles : vod

Exemple : http://example.com/encryption.key

Description :

L'URL d'un fichier de clé de déchiffrement à utiliser, avec une clé sous forme binaire (16 octets au total). Ceci est ignoré si un decryption_key est spécifié. 
  {
    "input": "s3://zencodertesting/test.mov",
    "decryption_key_url": "http://example.com/encryption.key"
  }

decryption_password

decryption_password:String

Versions de l'API : V1, V2

Parent : entrée

Valeurs valides : Un string

Types de tâches compatibles : vod

Exemple : mot de passe super secret

Description :

Mot de passe utilisé en combinaison avec la clé pour déchiffrer le fichier d'entrée.

  {
    "input": "s3://zencodertesting/test.mov",
    "decryption_password": "super-secret-password"
  }

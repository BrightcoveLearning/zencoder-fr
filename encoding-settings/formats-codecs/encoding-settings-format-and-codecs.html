format

format:String

Versions de l'API : V1, V2

Parent : sorties

Par défaut : Déterminé par le nom de fichier de sortie, puis par le codec vidéo ou audio. Sinon : mp4 (pour les sorties standard) ; ts (pour les sorties segmentées).

Valeurs valides : 3gp, aac, ac3, ec3, flv, m4f, mj2, mkv, mp3, mp4, mxf, ogg, ts, webm et wmv

Types de tâches compatibles : Tous

Exemple : webm

Description :

Le format de conteneur de sortie à utiliser.

Notez que certains conteneurs ont plusieurs extensions de fichier valides. En règle générale, des noms tels que « f4v », « ogv » et « m4v » sont utilisés pour les fichiers audio/vidéo, et « f4a », « m4a » et « oga » sont utilisés pour les fichiers audio uniquement, mais cela n'est pas appliqué. De même, "f4v" et "f4a" sont souvent utilisés pour le contenu mp4 conçu pour être lu dans Flash Video, mais ce n'est qu'une convention.

Compatibilité :

  • MP4 est compatible avec les vidéos h264 (par défaut), hevc et mpeg4, ainsi que l'audio aac (par défaut), mp3, ac3 et eac3. Valeurs valides et extensions de fichiers : mp4, m4a, m4v, f4v, f4a, m4b, m4r, f4b
  • m4f est compatible avec la vidéo h264 et hevc et l'audio AAC, ac3 et eac3.
  • ogg est compatible avec la vidéo theora et l'audio vorbis. Valeurs valides et extensions de fichiers : ogg, oga, ogv, ogx
  • webm est compatible avec la vidéo vp8 (par défaut) et vp9 et l'audio vorbis. Valeurs valides et extensions de fichiers : webm
  • 3gp est compatible avec les vidéos mpeg4 (par défaut) et h264, ainsi que l'audio aac (par défaut) et mp3. Valeurs et extensions de fichiers valides : 3gp, 3gp2, 3g2, 3gpp, 3gpp2. Notez que le format 3gp peut également utiliser une extension de fichier mp4.
  • flv est compatible avec les vidéos vp6 (par défaut) et h264, ainsi que l'audio aac (par défaut) et mp3. Valeurs valides et extensions de fichiers : flv
  • ts est compatible avec la vidéo h264 et l'audio aac (par défaut), mp3 ou ac3. Valeurs valides et extensions de fichiers : ts
  • mkv est compatible avec la vidéo h264 et l'audio AAC (par défaut), mp3, ac3 ou eac3. Valeurs valides et extensions de fichiers : mkv
  • WMV est compatible avec la vidéo WMV et l'audio WMA (par défaut) ou mp3. Valeurs valides et extensions de fichiers : wmv, wma
  • aac est compatible avec l'audio aac et aucune vidéo. Valeurs valides et extensions de fichiers : aac
  • mp3 est compatible avec l'audio mp3 et aucune vidéo. Valeurs valides et extensions de fichiers : mp3
  • ac3 est compatible avec l'audio ac3 et aucune vidéo. Valeurs valides et extensions de fichiers : ac3
  • ec3 est compatible avec l'audio eac3 et aucune vidéo. Valeurs valides et extensions de fichiers : ec3
  • mj2 est compatible avec la vidéo jp2 et l'audio aac. Valeurs valides et extensions de fichier : mj2, mjp2.
  • mxf est compatible avec la vidéo h264 et l'audio pcm. Valeurs valides et extensions de fichiers : mxf.
{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "format": "webm"
    }
  ]
}

video_codec

video_codec:String

Versions de l'API : V1, V2

Parent : sorties

Par défaut : Déterminé par le format, le profil ou le codec audio_codec. h264 si aucun n'est fourni.

Valeurs valides : h264, hevc, jp2, mpeg4, theora, vp6, vp8, vp9 et wmv

Types de tâches compatibles : Tous

Exemple : vp8

Description :

Codec vidéo de sortie à utiliser. Zencoder prend actuellement en charge les codecs suivants : h264 (par défaut), hevc, jp2, theora, vp6, vp8, vp9, mpeg4 et wmv.

Compatibilité :

  • h264 est compatible avec les formats audio aac (par défaut) et mp3 et mp4 (par défaut), m4f, flv ou 3gp
  • hevc est compatible avec les formats audio AAC (par défaut) ou mp3 et les formats mp4 ou m4f.
  • vp9 est compatible avec l'audio vorbis et le format webm
  • vp8 est compatible avec l'audio vorbis et le format webm
  • vp6 est compatible avec mp3 (par défaut) ou audio aac, et le format flv
  • theora est compatible avec l'audio vorbis et le format ogg
  • mpeg4 est compatible avec les formats audio aac (par défaut) et mp3, et les formats mp4 (par défaut) ou 3gp
  • wmv est compatible avec wma (par défaut) ou audio mp3, et le format wmv
  • jp2 est compatible avec l'audio PCM et le format mj2.
{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "video_codec": "vp8"
    }
  ]
}

audio_codec

audio_codec:String

Versions de l'API : V1, V2

Parent : sorties

Par défaut : Déterminé par le format, le profil ou le codec vidéo. aac si aucun n'est fourni.

Valeurs valides : aac, ac3, amr, eac3, mp3, pcm, vorbis et wma

Types de tâches compatibles : Tous

Exemple : mp3

Description :

Le codec audio à utiliser. Le codec par défaut dépend du codec vidéo et du format sélectionnés.

Notez que l'audio MP3 dans un conteneur MP4 est valide avec le contenu vidéo H.264, mais n'est pas lu dans Quicktime.

Compatibilité :

  • aac est compatible avec la vidéo h264 (format mp4, m4f, 3gp ou flv), la vidéo mpeg4 (format mp4 ou 3gp) et vp6 (format flv)
  • vorbis est compatible avec la vidéo vp8 (format webm) et la vidéo theora (format ogg)
  • wma est compatible avec la vidéo wmv et le format wmv
  • mp3 est compatible avec la vidéo h264 (format mp4, 3gp ou flv) et la vidéo wmv (format wmv). mp3, h264 et mp4 ne seront pas lus dans Quicktime
  • pcm est compatible avec la vidéo h264 et le format mxf.
{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "audio_codec": "mp3"
    }
  ]
}

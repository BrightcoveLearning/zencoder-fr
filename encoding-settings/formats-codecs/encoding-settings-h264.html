Recommandations d'encodage

En plus des paramètres trouvés sur cette page, vous souhaiterez peut-être inclure les paramètres vidéo généraux suivants.

  • video_reference_frames
  • video_codec_profile
  • video_codec_level
  • video_bframes

tuning

tuning:String

Versions de l'API : V1, V2

Parent : sorties

Valeurs valides : film, animation, grain, psnr, ssim, fastdecode ou zérolatence

Types de tâches compatibles : Tous

Exemple : film

Description :

Réglez la vidéo de sortie pour l'optimiser pour un type de contenu spécifique ou une priorité d'encodage spécifique.

En coulisses, cela définit l'option x264 —tune.

Valeurs acceptées :

  • film - optimisé pour la plupart des contenus vidéo non animés (pas seulement les longs métrages).
  • animation - optimisée pour l'animation. Notez que la plupart des animations 3D se comportent plus comme des films et non comme des animations dessinées à la main, alors utilisez-les uniquement pour les animations dessinées à la main (anime, Disney classique, etc.).
  • grain - optimisé pour les films avec des niveaux élevés de grain.
  • psnr - utilise le « rapport signal sur bruit de crête » pour optimiser la qualité vidéo.
  • ssim - utilise la « similitude structurelle » pour optimiser la qualité vidéo.
  • fastdecode - réduit la complexité de l'encodage pour faciliter le décodage.
  • zéro latency - x264 conservera un tampon interne de trames pour améliorer la qualité ; ce paramètre supprime ce tampon, mais réduit la qualité.
{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "tuning": "film"
    }
  ]
}

crf

crf:Integer

Versions de l'API : V1, V2

Parent : sorties

Valeurs valides : 1-51

Types de tâches compatibles : Tous

Exemple : 26

Description :

CRF est un paramètre de contrôle binaire et définit un facteur de vitesse constant. Ceci est étroitement lié à l'encodage de quantificateur constant, bien que le mode CRF obtient généralement une meilleure compression que le codage Q constant en réduisant la qualité du quantificateur sur les trames « moins importantes ». En savoir plus sur crf.

Le paramètre Zencoder Quality détermine un CRF approprié pour une vidéo donnée. Donc, si vous définissez à la fois Qualité et CRF, le paramètre CRF remplacera Qualité.

Les valeurs valides sont 0-51, les valeurs inférieures étant de meilleure qualité. En général, vous ne devriez pas avoir à passer en dessous de 16 ; 16 est presque sans perte pour la plupart des fichiers. Autour 24 semble généralement assez bon. Autour 35 semble assez comprimé.

  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "crf": "26"
    }
  ]
}

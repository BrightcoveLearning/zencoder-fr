    Paramètres de traitement audio

    Cette rubrique contient des détails sur les paramètres d'encodage liés au traitement audio. Pour d'autres paramètres spécifiques à l'audio, voir l'audio et Contrôle de taux.

    audio_gain

    audio_gain:Float

    Versions de l'API: V1, V2

    Parent: les sorties

    Valeurs valides: Un nombre entre -60 et 60

    Types de travaux compatibles: Tous

    Exemple: 5

    Description :

    Appliquez une quantité de gain à l'audio. Cela augmente ou diminue le volume de l'audio de la quantité spécifiée après l'application d'autres effets. Ceci est principalement utile pour ajuster les niveaux après avoir appliqué d'autres effets ou pour effectuer des ajustements sur des fichiers dont vous connaissez déjà le niveau. Les sons qui dépassent légèrement le volume maximum seront légèrement limités pour préserver la fidélité, mais une augmentation significative au-dessus du volume maximum entraînera une distorsion du son. Voir audio_normalize pour augmenter automatiquement le volume sans distorsion. Spécifié en dB, positif ou négatif, jusqu'à 60 dB.

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "audio_gain": 5
    }
  ]
}

    Voir également: audio_normalize

    audio_normaliser

    audio_normalize:Boolean

    Versions de l'API: V1, V2

    Parent: les sorties

    Défaut: faux

    Valeurs valides: vrai ou faux

    Types de travaux compatibles: vod

    Exemple: vrai

    Description :

    Normaliser l'audio à 0dB. Cela augmente le volume de l'audio autant que possible sans provoquer de distorsion. Pour assurer un ajustement cohérent par les effets d'expansion et de compression, nous normalisons le volume avant et après ces effets. Voir audio_pre_normalize et audio_post_normalize pour ne normaliser qu'à l'un de ces points. audio_normalize équivaut à spécifier à la fois audio_pre_normalize et audio_post_normalize.

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "audio_normalize": true
    }
  ]
}

    Voir également: audio_gain , audio_pre_normalize , et audio_post_normalize

    audio_passe-haut

    audio_highpass:Integer

    Versions de l'API: V1, V2

    Parent: les sorties

    Valeurs valides: Une fréquence en Hz, de 5 à 24000

    Types de travaux compatibles: Tous

    Exemple: 10

    Description :

    Appliquez un filtre passe-haut à l'audio à la fréquence spécifiée en Hz. Cela empêchera les fréquences audio inférieures au seuil spécifié d'être incluses dans la sortie. Une utilisation très courante de ce paramètre est de réduire les grondements / bruits audio, tels que les sons des camions qui passent à l'extérieur pendant l'enregistrement ou certains sons provoqués par le vent ou frappant un microphone. DC-Shift peut également être réduit / supprimé en spécifiant un filtre passe-haut à une basse fréquence, telle que 10 Hz.

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "audio_highpass": 10
    }
  ]
}

    Voir également: audio_lowpass

    audio_passe-bas

    audio_lowpass:Integer

    Versions de l'API: V1, V2

    Parent: les sorties

    Valeurs valides: Une fréquence en Hz, de 5 à 24000

    Types de travaux compatibles: Tous

    Exemple: 3 000

    Description :

    Appliquez un filtre passe-bas à l'audio à la fréquence spécifiée en Hz. Cela empêchera les fréquences audio supérieures au seuil spécifié d'être incluses dans la sortie. Cela peut être utilisé pour couper les sons de gémissement très aigus de l'audio, ou pour créer des effets spéciaux, comme un réglage «téléphone» (en utilisant audio_highpass de 300 et audio_lowpass de 3000).

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "audio_lowpass": 3000
    }
  ]
}

    Voir également: audio_highpass

    audio_compression_ratio

    audio_compression_ratio:Float

    Versions de l'API: V1, V2

    Parent: les sorties

    Valeurs valides: 1,0 à 30,0

    Types de travaux compatibles: Tous

    Exemple: 2.5

    Description :

    Compresser la plage dynamique de l'audio en réduisant le volume à un rapport spécifié de N: 1, au-dessus de la valeur spécifiée audio_compression_threshold. Les rapports entre 1,0 et 30,0 sont valides. La compression est essentiellement utile pour rendre le volume audio plus cohérent entre les points faibles et les points forts. Cela peut être utilisé pour rendre les spots bruyants (crier, portes claquées, etc.) moins saccadés. Utilisé en conjonction avec audio_gain ou audio_normalization, cela peut rendre les parties calmes d'un discours plus audibles.

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "audio_compression_ratio": 2.5
    }
  ]
}

    Voir également: audio_compression_threshold

    audio_compression_threshold

    audio_compression_threshold:Float

    Versions de l'API: V1, V2

    Parent: les sorties

    Défaut: -20

    Valeurs valides: Un nombre négatif entre 0 et -120

    Types de travaux compatibles: Tous

    Exemple: -30

    Description :

    Définissez le seuil au-dessus duquel la compression audio est appliquée, lorsque audio_compression_ratio est spécifié. Les valeurs sont spécifiées en dB sous la forme d'un nombre négatif jusqu'à -120.

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "audio_compression_threshold": -30
    }
  ]
}

    Voir également: audio_compression_ratio

    audio_expansion_ratio

    audio_expansion_ratio:Float

    Versions de l'API: V1, V2

    Parent: les sorties

    Valeurs valides: 1,0 à 30,0

    Types de travaux compatibles: Tous

    Exemple: 2.5

    Description :

    Élargissez la plage dynamique de l'audio en réduisant le volume à un rapport spécifié de N: 1, en dessous du audio_expansion_threshold. Les rapports entre 1,0 et 30,0 sont valides. L'expansion est essentiellement utile pour réduire les sons indésirables pendant les points calmes ou pour rendre plus significative la différence entre les sons forts et faibles.

    {

  "outputs": [
    {
      "audio_expansion_ratio": 2.5
    }
  ]
}

    Voir également: audio_expansion_threshold

    audio_loudness_level

    audio_loudness_level:Float

    Versions de l'API: V1, V2

    Parent: les sorties

    Valeurs valides: Un certain nombre de décibels, de -60,0 à 60,0

    Types de travaux compatibles: vod

    Exemple: -23

    Description :

    Réglez le niveau sonore de l'audio. Ceci est mesuré en LUFS et spécifié en dB. Ceci est utile pour régler le niveau de sonie de sortie pour se conformer à une norme (-23 dB pour EBU R.128)

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "audio_loudness_level": -23
    }
  ]
}

    audio_karaoke_mode

    audio_karaoke_mode:Boolean

    Versions de l'API: V1, V2

    Parent: les sorties

    Défaut: faux

    Valeurs valides: vrai ou faux

    Types de travaux compatibles: Tous

    Exemple: vrai

    Description :

    Appliquez un effet «karaoké» à l'audio en soustrayant le canal audio droit de la gauche, et en ne gardant que la différence. S'applique uniquement aux sorties stéréo.

    Soyez prudent en utilisant cette fonctionnalité:

    • Cela produit parfois de la distorsion
    • Ça ne fait parfois rien du tout
    • Cela affecte parfois négativement le bruit de fond ou la musique
    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "audio_karaoke_mode": true
    }
  ]
}

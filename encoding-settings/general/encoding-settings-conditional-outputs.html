min_size

min_size:String

Versions de l'API : V2

Parent : conditionals

Types de tâches compatibles : Tous

Exemple :

  • 640 x 480

Description :

Si les dimensions de la source sont inférieures à la taille demandée, la sortie sera ignorée.

Les dimensions sont calculées en multipliant la hauteur et la largeur de la source et en les comparant au produit des dimensions fournies.

Note : En raison de la nature des métadonnées vidéo et des incohérences entre les types de vidéo, certains fichiers source peuvent voir leurs dimensions mal lues lors de l'inspection, ce qui entraîne l'application ou l'ignorance des conditions.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "skip": {
        "min_size": "640x480"
      }
    }
  ]
}

min_durée

min_duration:Integer

Versions de l'API : V2

Parent : conditionals

Valeurs valides : Un entier positif

Types de tâches compatibles : Tous

Exemple :

  • 300

Description :

Si la durée de la source est plus courte que la durée demandée, la sortie sera ignorée.

La durée doit être fournie en secondes. Pour ignorer une sortie si la source est inférieure à 5 minutes, utilisez « 300 ».

Note : En raison de la nature des métadonnées vidéo et des incohérences entre les types de vidéo, la durée de certains fichiers source peut être lue de manière incorrecte lors de l'inspection, ce qui entraîne des conditions appliquées ou ignorées de manière incorrecte.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "skip": {
        "min_duration": "300"
      }
    }
  ]
}

Voir aussi : max_duration

min_audio_débit

min_audio_bitrate:Integer

Versions de l'API : V2

Parent : conditionals

Valeurs valides : Un entier positif

Types de tâches compatibles : Tous

Exemple :

  • 128

Description :

Si le débit binaire audio de la source est inférieur au débit binaire spécifié (en kbps), la sortie sera ignorée.

Les fichiers source sans piste audio ou sans débit audio reconnaissable ignoreront ce paramètre. (Voir require_audio pour plus d'options.) Pour sauter une sortie si le débit binaire audio source est inférieur à 128 Kbps, utilisez 128.

Note : En raison de la nature des métadonnées audio et des incohérences entre les types audio, certains fichiers source peuvent voir leurs débits binaires lus de manière incorrecte lors de l'inspection, entraînant des conditions appliquées ou ignorées de manière incorrecte.

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "skip": {
        "min_audio_bitrate": 128
      }
    }
  ]
}

Voir aussi : max_audio_bitrate et require_audio

require_audio

require_audio:Boolean

Versions de l'API : V2

Parent : conditionals

Valeur par défaut : false

Valeurs valides : vrai ou faux

Types de tâches compatibles : Tous

Exemple : true

Description :

Si le fichier source n'a pas de piste audio, la sortie sera ignorée.

Pour ignorer une sortie si la source n'a pas de piste audio, définissez cette option sur true.

Note : En raison de la nature des métadonnées multimédia et des incohérences entre les types de supports, certains fichiers source peuvent voir leurs métadonnées lues de manière incorrecte lors de l'inspection, ce qui entraîne des conditions appliquées ou ignorées de manière incorrecte.


  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "skip": {
        "require_audio": true,
        "min_audio_bitrate": 64
      }
    }
  ]
}

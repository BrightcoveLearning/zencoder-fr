Signification

Cette erreur signifie que le CDN a rejeté un flux RTMP car il y a trop de flux utilisant le même ID de flux. Ainsi, le nombre maximal de flux par ID est dépassé.

Un complément d'information

Cela peut signifier :

  • Le nombre maximal de flux par ID configuré dans le CDN est trop faible pour accepter tous les flux demandés
  • D'autres tâches Zencoder sont déjà diffusées en continu vers ce CDN et ID de flux
  • L'ID de flux CDN est utilisé par un autre système en dehors de Zencoder
  • Le CDN pourrait avoir des' flux de zombies, où un flux précédent n'a pas été correctement déconnecté et compte toujours dans la limite maximale de flux pour cet ID.

Pour éviter ce problème, vous pouvez arrêter les autres tâches qui ont des flux utilisant cet ID de flux, ou utiliser un autre ID de flux si vous l'avez. Pour plus d'informations, vous pouvez Contactez le support avec le Job ID et notre équipe d'ingénieurs enquêtera

Réessayer ce boulot ?

Emplois avec cette erreur ne devrait pas être retenté. En savoir plus sur la relance des tâches.

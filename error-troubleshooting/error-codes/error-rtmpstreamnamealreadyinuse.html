Signification

Cette erreur signifie que le CDN a rejeté un flux RTMP car le nom du flux est déjà utilisé par un autre flux. En d'autres termes, deux encodeurs RTMP tentent de pousser vers le même point d'entrée CDN, et le second est rejeté parce que le premier est déjà en cours.

Un complément d'information

Cela peut signifier :

  • Un autre travail Zencoder est déjà en cours de diffusion vers ce nom de flux CDN
  • Le nom du flux CDN est utilisé par un autre système en dehors de Zencoder
  • Une tâche précédente n'a pas pu se déconnecter du CDN pour une raison quelconque

Pour éviter ce problème, vous pouvez simplement utiliser un nom de flux CDN différent, ou mettre fin à tous les travaux précédents en utilisant le même nom de flux. Si un travail ne se termine pas pour une raison quelconque, Contactez le support avec le Job ID et notre équipe d'ingénieurs enquêtera

Réessayer ce boulot ?

Emplois avec cette erreur ne devrait pas être retenté. En savoir plus sur la relance des tâches.

Codes d'erreur

Cette rubrique fournit un résumé des codes d'état HTTP pour les requêtes d'API Zencoder et un index des codes d'erreur renvoyé par l'API Zencoder. Cliquez sur l'un des codes d'erreur pour voir plus de détails.

Codes d'état HTTP

Le tableau suivant répertorie le code de réponse HTTP pour les requêtes API Zencoder, dans le cas d'erreurs, indique la cause la plus probable, dans la mesure du possible.

Codes de réponse HTTP
Code d'état Explication probable
200 OK Réponse de réussite standard pour les actions qui ne créent pas de nouvel actif
201 CREATED Réponse standard pour les demandes qui créent de nouvelles ressources
204 NO CONTENT Réponse standard pour une requête qui ne renvoie aucune réponse
400 BAD REQUEST JSON non valide dans le corps de la requête
401 UNAUTHORIZED Clé API manquante ou non valide
402 PAYMENT REQUIRED Retourné lorsqu'une requête API tente de désactiver le mode d'intégration Zencoder https://app.zencoder.com/docs/api/accounts/integration sur un compte où la facturation n'a pas été configurée. Vous devriez aller à www.zencoder.com et choisissez un plan de paiement
403 FORBIDDEN limite de taux pour les demandes dépassées, ou en réponse à une demande d'API de relance / resoumettre pour les ressources qui ont été réessayées trop souvent
404 NOT FOUND URL de demande incorrecte ou ID de travail invalid/ancien
406 NOT ACCEPTIBLE Retourné pour tentative de création d'informations d'identification non-AWS via l'API
409 CONFLICT Généralement seulement lorsque vous essayez de réessayer/soumettre à nouveau un travail qui est trop vieux
422 UNPROCESSIBLE ENTITY Retourné lorsqu'une requête API ne peut pas être validée
500 INTERNAL SERVER ERROR Soit un problème temporaire avec Zencoder ou quelque chose de mal avec la requête
503 SERVICE UNAVAILABLE Zencoder temporairement indisponible, probablement pendant une maintenance planifiée
504 GATEWAY TIMEOUT Retourné si le service Zencoder est en panne

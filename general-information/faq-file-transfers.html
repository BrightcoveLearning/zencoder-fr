FAQ : Transferts de fichiers

Cette rubrique répond aux questions sur le transfert de fichiers vers Zencoder pour l'encodage et vers vos destinations pour les sorties.

FAQ

  • Quels protocoles de transfert prendrez-vous en charge ? HTTP ? FTP ?

    Entrée : nous prenons en charge HTTP, HTTPS, FTP, SFTP, Rackspace Cloud Files, Google Cloud Storage, Azure et Amazon S3.

    Sortie : nous prenons en charge FTP, SFTP, Rackspace Cloud Files, Azure, Google Cloud Storage et Amazon S3. Nous ne prenons actuellement pas en charge les téléchargements HTTP généraux.

    Utilisateurs de Cloud Files : Vous pouvez spécifier la région à utiliser (DFW, ORD ou UK) en l'ajoutant au protocole, comme cf+ord://username:api_key@container/object. Actuellement, les seules régions Cloud Files prises en charge sont DFW, ORD et UK. La région DFW sera utilisée par défaut.

    Besoin d'une autre méthode de transfert ? Faites le nous savoir.

  • Où puis-je placer mes fichiers originaux pour le transcodage ?

    Nous pouvons extraire des fichiers de n'importe où via HTTP, HTTPS, FTP ou SFTP. Nous pouvons également extraire des fichiers d'un conteneur Azure, de Google Cloud Storage ou d'un bucket S3, y compris des buckets privés, et de Rackspace Cloud Files.

    Exemples :

    http://example.com/path/to/file.avi
http://s3.amazonaws.com/bucket/file.avi
s3://bucket/file.avi
gcs://bucket/file.avi
cf://username:api_key@container/file.avi
azure://account-name:account-key@container/file.avi
https://example.com/path/to/file.avi
ftp://example.com/path/to/file.avi
sftp://example.com/path/to/file.avi
    Remarque sur les ports : si vous devez inclure un port dans l'adresse du fichier, il doit être déclaré à la fin du domaine dans l'URL, comme ceci : 
    sftp://example.com:8888/path/to/file.avi

  • Pouvez-vous transférer des fichiers en toute sécurité ?

    Sûr. Utilisez SFTP ou HTTPS et incluez un nom d'utilisateur et un mot de passe dans l'URL, comme ceci :

    
https://user:pass@example.com/file.mp4
sftp://user:pass@example.com/file.mp4

  • Pouvez-vous extraire des fichiers à partir d'un compartiment privé S3, Google Cloud Storage ou Cloud Files ?

    Oui. Vous pouvez ajouter un accès identifiants à votre compte Zencoder, ou accorder un accès en lecture au bucket pour Zencoder.

  • Y a-t-il des limites de taille de fichier ?

    Non. Nous n'avons aucune limite de taille de fichier et, en fait, nous avons choisi de ne pas facturer de frais supplémentaires pour les vidéos de grande taille afin d'encourager les clients à nous envoyer leurs meilleures copies. Plus la qualité des vidéos que vous nous envoyez est élevée, plus la qualité de sortie que nous pouvons vous offrir est élevée.

    Il y a, cependant, une limite de 24 heures sur la durée de la vidéo source.

  • Où vont les vidéos finies ?

    Vous avez quelques options pour savoir où nous enverrons les vidéos.

    Nous pouvons envoyer les fichiers vers un serveur FTP ou SFTP, vers un bucket S3 ou Google Cloud Storage ou vers Rackspace Cloud Files. Voir le option url dans nos documents API pour plus d'informations.

    N'oubliez pas d'ajouter identifiants ou donnez à Zencoder l'accès à votre seau si vous utilisez S3 ou Google Cloud Storage.

    Vous pouvez également ne spécifier aucune destination de sortie et nous conserverons le fichier pendant 24 heures. Nous vous fournirons une URL que vous pouvez utiliser pour télécharger la vidéo. Nous supprimerons la vidéo après 24 heures, alors n'oubliez pas de la télécharger.

    À partir de la version 2 de l'API, si nous ne sommes pas en mesure de télécharger des fichiers sur votre serveur, nous les téléchargerons à la place sur S3 et vous fournirons une URL pour tous les fichiers.

  • Comment définir les autorisations ACL si j'envoie des fichiers codés vers S3 ?

    Utilisez notre API. Consultez la documentation de l'API sur contrôle d'accès pour plus d'informations.

    • Transferts de fichiers

    Zencoder fonctionne sur des instances Amazon EC2, il existe donc une large gamme d'adresses IP que nous utilisons pour le transcodage de fichiers et en direct. Amazon ajoute régulièrement de nouvelles plages à la liste, qui peuvent être trouvées via AWS API de plage IP.

    Notifications

    Les notifications sont actuellement envoyées depuis sg-77f03012, mais elles peuvent changer à tout moment. Afin de tenir compte de cela, nous avons ajouté un X-Zencoder-Notification-Secret en-tête à toutes les notifications livrées. Cela peut être trouvé sur le API tableau de bord, où vous pouvez voir votre secret actuel et en générer un nouveau si nécessaire.

    Cet en-tête est livré avec chaque demande et peut être utilisé pour garantir que la notification a été envoyée depuis votre compte Zencoder. Remarque: Afin de protéger vos notifications secrètes, veillez à envoyer des notifications aux points de terminaison HTTPS !

    Note : Les serveurs peuvent migrer vers de nouveaux centres de données et les adresses IP peuvent changer sans préavis.

  • Puis-je expédier un disque dur avec mes vidéos à Zencoder ?

    Si vous avez un grand nombre de vidéos à encoder mais que vous ne souhaitez pas les transférer sur Internet, vous pouvez utiliser Amazon Importation / exportation AWS. Vous expédierez un disque dur avec vos vidéos à Amazon, qui les téléchargera sur S3 pour vous.

    Amazon a instructions simples pour préparer votre disque à expédier, où l'envoyer, etc.

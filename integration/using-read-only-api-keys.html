Vue d'ensemble

Les requêtes API en lecture seule sont utiles lorsque vous effectuez des requêtes JSONP sans exposer une clé API pouvant créer de nouvelles tâches sur votre compte.

Demandes d'API compatibles

Les requêtes suivantes sont compatibles avec une clé API en lecture seule.

  • Détails du poste
  • Progression du travail
  • Détails d'entrée
  • Progression d'entrée
  • Détails de sortie
  • Progression de la sortie

Exemples de désinfection

Les demandes effectuées à l'aide de clés d'API en lecture seule auront certaines informations supprimées de la réponse pour protéger les informations sensibles. Cela inclut, mais sans s'y limiter, les URL (dans le cas où des informations sur le nom d'utilisateur ou le mot de passe sont fournies), des messages d'erreur spécifiques et d'autres informations potentiellement sensibles.

Rappels JSONP

Pour AJAX et d'autres requêtes publiques, les clés API en lecture seule peuvent accéder à ces informations à l'aide de JSONP en demandant l'URL avec une extension .js. La réponse utilisera le progress rappel sauf si un est spécifié en utilisant le callback dans la chaîne de requête. Par exemple, une demande JSONP pour la progression des entrées sans rappel spécifié :

progress(
  {
      "state": "processing",
      "current_event": "Downloading",
      "progress": "32.34567345"
  }
)
